// 01 — capabilities

Everything you need to
test confidently

⬡

Page Object Model

Clean separation of locators, page actions, and test logic. Maintainable, readable, and DRY.

◈

Allure Reporting

Rich HTML reports with step-by-step breakdowns, severity levels, and automatic failure screenshots.

⊕

Auto Screenshots

On test failure, a PNG screenshot is captured and attached to the Allure report for instant debugging.

⊞

Config-driven

Browser type and target URL are externalized in config.ini for easy environment switching.

◐

Pytest Markers

Tests tagged with smoke and custom IDs like testId001 for selective execution.

⊛

Extensible API Module

An API/ package is scaffolded and ready for REST API test cases — grow as you go.

@pytest.hookimpl(tryfirst=True, hookwrapper=True)
def pytest_runtest_makereport(item, call):
    outcome = yield
    rep = outcome.get_result()
    if rep.when == "call" and rep.failed:
        driver = item.funcargs.get("driver")
        if driver:
            allure.attach(driver.get_screenshot_as_png(),
                name="failure_screenshot",
                attachment_type=allure.attachment_type.PNG)

import pytest
import allure

@allure.feature("Authentication")
@allure.story("Login")
@allure.severity(allure.severity_level.CRITICAL)
@pytest.mark.smoke
@pytest.mark.testId001
def test_valid_login(driver):
    """Valid credentials should navigate to Dashboard."""

    login_page = LoginPage(driver)

    with allure.step("Navigate to login page"):
        login_page.open()

    with allure.step("Enter valid credentials"):
        login_page.enter_username("Admin")
        login_page.enter_password("admin123")
        login_page.click_login()

    with allure.step("Assert Dashboard is visible"):
        assert login_page.is_dashboard_visible()

// 03 — getting started

Up and running
in minutes

1

Clone the repository

Grab the code and move into the project directory.

git clone https://github.com/supriya-mainalli/pytest-automations-e2e.git
cd pytest-automations-e2e
2

Create a virtual environment

Isolate dependencies to keep things clean.

python -m venv venv
source venv/bin/activate        # macOS / Linux
venv\Scripts\activate           # Windows
3

Install dependencies

Pytest, Selenium, and the Allure adapter.

pip install pytest selenium allure-pytest
4

Configure target browser & URL

Edit WebAutomations/config.ini to point at your environment.

[basic info]
browser  = chrome
endpoint = https://opensource-demo.orangehrmlive.com/web/index.php/auth/login

// 06 — allure reporting

Every test tells a
complete story

Each test is annotated with metadata that Allure uses to generate rich, navigable HTML reports. Failures include browser screenshots automatically.

@allure.feature

Feature grouping

Groups tests by high-level product area for easy navigation.

@allure.story

User story

Maps each test to a user-facing scenario or acceptance criterion.

@allure.severity

Severity level

CRITICAL, NORMAL, MINOR — filter by risk when time is short.

allure.step()

Step breakdown

Granular step-by-step view in the report — pin down exactly where failures occur.
